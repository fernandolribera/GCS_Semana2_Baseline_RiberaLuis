# GCS_Semana2_Baseline_RiberaLuis

# Objetivo
Este proyecto tiene como objetivo aprender y aplicar la Gestión de la Configuración del Software (GCS), usando buenas prácticas como el control de versiones, la creación de líneas base, la gestión de cambios y la trazabilidad, apoyándose en herramientas como Git.

## Estructura del repositorio
- `/docs`: Contiene la documentación del proyecto.
- `/src`: Contiene el código fuente.
- `/tests`: Contiene los casos de prueba.
- `/config`: Archivos de configuración.
- `/scripts`: Scripts de despliegue y otros.
